计算机网络的初识

发表于 2021-03-12

前言

To be honest，作为一个计算机专业的学生，我对计算机一点也不了解。我不知道什么是DNS，什么是网络带宽，是一个完完全全一窍不通的白中白，所以希望我能学的好吧^^

大体介绍

计算机网络始于20世纪50年代，在我们现在用的计算机之前一共经历了四个阶段，产生巨大影响的是 阶段二 和 阶段三

阶段二

主要是产生了计算机网络，由ARPANET研究提出了

  1. 资源子网和通信子网的两个概念
  2. 研究了报文 分组交换 的数据交换方法
  3. 采用了层次化的网络体系结构模型和协议体系
  4. 促进 IP/TCP 协议的发展

啥是IP/TCP协议

在维基百科上我找到了这样的解释

互联网协议套件（英语：Internet Protocol Suite，缩写IPS）[1]是网络通信模型，以及整个网络传输协议家族，为网际网络的基础通信架构。它常通称为TCP/IP协议族（英语：TCP/IP Protocol Suite，或TCP/IP Protocols），简称TCP/IP[2]。因为该协议家族的两个核心协议：TCP（传输控制协议）和IP（网际协议），为该家族中最早通过的标准[3]。由于在网络通讯协议普遍采用分层的结构，当多个层次的协议共同工作时，类似计算机科学中的堆栈，因此又称为TCP/IP协议栈（英语：TCP/IP Protocol Stack）

但是我觉得比较能够理解的是下面这句话

TCP/IP提供了点对点链接的机制，将资料应该如何封装、寻址、传输、路由以及在目的地如何接收，都加以标准化。它将软件通信过程抽象化为四个抽象层，采取协议堆栈的方式，分别实现出不同通信协议。

不过学到现在我还是不太能理解，所以等到学到后面我理解了之后会重新修改 XD

阶段三

主要是网络体系结构的标准化
我理解的是：每个公司要完成产品的互联互通，必须有一个标准，于是乎每个公司都有了自己的标准。但是这太过于局限了，所以 国际标准化 ISO 组织提出了 OSI 标准，各个公司只要遵循这个标准，就真正意义上实现了互联互通。

关于计算机网络的定义

  1. 广义的观点
  2. 资源共享的观点
  3. 用户透明性的观点

资源共享的观点

第一条我觉得很重要

  1. 建立的目的是实现资源共享
  2. 计算机之间没有主从关系，每台计算机都是独立的
  3. 必须遵行共同的协议

资源共享的基础

基础是数据通信

网络的构成

网络由若干的 节点(Node) 和 链路(Link) 组成

计算机网络的逻辑结构

负责处理的 主机 和 终端

互联网的网络好比云，咱们的主机便是因特网的边缘部分，也就是资源子网，云就是因特网的核心部分，也就是通信子网。

核心部分

核心部分由大量网络和连接这些网络的路由器组成。为边缘部分提供服务的(提供连通性和交换)

啥是分组交换

就是对资源进行动态分配

电路交换的过程
  1. 建立连接
  2. 交换数据
  3. 释放数据
电路交换的特点
  1. 在数据交换前建立一条从发端到收端的 物理通路，
  2. 在数据交换的全部时间内用户始终占用端到端的固定传输信道
  3. 交换双方可实时进行数据交换而无延迟
电路交换的缺点
  1. 传输效率低
  2. 数据交换往往有突发性和间歇性特征
  3. 无灵活性
  4. 任何一方出现故障要重新连接

因此用分组交换来完成
分组交换能够通过当前状态进行动态分配

边缘部分

边缘部分由所有连接在因特网上的主机组成，用户直接使用的，用来进行通信和资源共享

小问题

网络应用是主机还是程序呢？

网络应用是程序。

客户服务器的方式

俗称 C/S 方式(Client/Server)

是最常用最传统的方式，我们访问网站，发邮件都是这个方式。由客户端发起请求，服务器接受请求，我们便能获得自己想要的结果。

对等连接的方式

俗称 P2P 方式。

只要两个主机都运用了P2P软件，它们就能直接交换信息来共享计算机的资源和服务。不区分服务请求方或者是服务提供方 ，每个主机既是客户也是服务器。主要用于迅雷等软件下载和上传文件。

网络的分类

  1. 局域网(LAN)
  2. 广域网(WAN)
  3. 城域网(MAN)
  4. 接入网(AN)

局域网(LAN)

优势

  1. 规模小
  2. 硬件设备相对简单
  3. 由于距离近传输速率高，误码率相对低
  4. 组建方便，技术较简单

广域网(WAN)

涉及范围大，广，因为传输较远，所以速率较低，误码率高于局域网，控制机制复杂

接入网(AN)

又称本地接入网和居民接入网，一般由ISP提供，用户与因特网连接的桥梁。

城域网(MAN)

介于局域网和广域网之间，很多城市采用的是以太网技术。

Java概述

  Java是一个面对对象的语言，这里我们就要讨论一下面对对象和面对过程的具体细节。它们分别是什么含义。

面对对象

  我理解的面向对象是指分开来完成，将程序分为几个板块进行实现。我在网上搜寻的过程中看见有一位大神将面向对象比喻成“做包子”，将包子皮专门做一堆，将包子馅做另外一堆，最后将二者一并完成。

面对过程

  以过程为核心，将程序进行一步步执行。这也就是每做一个包子时，都会按照一套程序顺序进行，不会出现单独做包子皮或者单独做包子馅的现象

  以上的理解如果有误还请纠正我。
